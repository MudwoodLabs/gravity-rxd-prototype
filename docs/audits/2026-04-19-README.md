# Security Audit — 2026-04-19

Five parallel reviewers examined the prototype shortly after it was made
public. Each report below has a different focus. Findings overlap in places;
cross-reference by the tags in square brackets.

## Reports

- [`01-owasp-secrets-supply-chain.md`](./01-owasp-secrets-supply-chain.md) — OWASP / secrets / deps / CLI UX
- [`02-bitcoin-spv-crypto-correctness.md`](./02-bitcoin-spv-crypto-correctness.md) — Bitcoin-side crypto correctness (witness stripping, Merkle proofs, PoW math, RNG)
- [`03-radiantscript-covenant.md`](./03-radiantscript-covenant.md) — RadiantScript covenants (opcode semantics, arithmetic bounds, state machine, MINIMALDATA)
- [`04-atomic-swap-economic-security.md`](./04-atomic-swap-economic-security.md) — Economic / game-theoretic attacks (griefing, fee-sniping, binding, recovery)
- [`05-spv-data-integrity.md`](./05-spv-data-integrity.md) — Data integrity across the SPV pipeline (mempool.space → relayer → covenant)

## Synthesis — Show-stoppers

| # | Area | Finding | Attack cost |
|---|------|---------|-------------|
| CV-1 | Covenant | No bound on `nBits` target; anchor provides no protection without it | hours of CPU, ≪$1 |
| CV-2 | Covenant | Attacker-chosen `outputOffset` with no structural tx-parse; embed pattern in OP_RETURN | ~1 BTC tx fee |
| CV-3 | Covenant | Committed `.rxd` files stale vs generator — weaker than docs claim | N/A — correctness |
| REL-1 | Relayer | Shell injection in `broadcast --method ssh` (default path) | one malicious tx-hex file |
| REL-2 | Relayer | `--privkey-wif` on argv — documented as standard flow | local user observation |
| ECON-1 | Economic | `claimDeadline=0` default — forfeit/finalize race from block 1 | 1 Radiant tx fee |

| INT-2 | Integrity | Relayer never aligns `h1` with the chain anchor | N/A — correctness |

## Synthesis — High

- BTC: 64-byte-tx Merkle-collision defense missing
- CV: two `extract_*_code_hash.js` helpers disagree — wrong pick locks funds
- CV: `maker_claimed.rxd` is a STUB with no SPV check (total loss if deployed)
- INT: `extract_code_hash.js::indexOf(0xbd)` breaks for ~8% of Taker pkhs
- ECON: `claim()` is permissionless + unsigned — third-party can advance state
- SEC: `refs/original/*` + backup tag still contain the scrubbed email locally
- SEC: hard-coded `/home/eric/...` in `validation/build_cancel_tx.js:23`
- SEC: personal paths still in `HANDOFF.md:53`, `GRAVITY_ANALYSIS.md:253`
- INT: mempool.space single-point-of-trust with no fallback

## Assessment

- Solo-operator demo / research prototype: fine. Primitives work; the mainnet
  trade is real.
- Bilateral trade with a real counterparty: `claimDeadline=0` plus
  permissionless `claim()` lets either side grief. Don't.
- Public deployment: CV-1 and CV-2 each independently allow any attacker to
  drain any Maker UTXO. The paper's economic analysis does not reflect the
  code as deployed.

## Fix sequence

1. **Docs-only, immediate:** safety banner in README, personal-info cleanup,
   `SECURITY.md`, `refs/original/*` purge.
2. **Relayer (~1 day):** kill shell injection, `--privkey-file`, wire reference
   validators, fix anchor alignment, fix `indexOf(0xbd)` walker.
3. **Covenant (~1 week):** `nBits` bound, structural tx-parse, Taker sig on
   `claim()`, non-zero `claimDeadline`, regenerate committed `.rxd`.
4. Re-audit, then re-run mainnet trade with a different counter-party.
